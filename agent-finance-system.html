Agent智能财务系统

这是我基于 Codex 独立推进的智能财务系统项目，围绕“费控、报销、付款、凭证、电子档案、Agent 自动化”构建统一的财务协同平台，目标是把原本分散在 Excel、邮件、ERP 和线下流转中的动作连接成一条可追踪、可配置、可持续迭代的数字化链路。

Vue 3 + Router + Pinia Element Plus Spring Boot + Maven MySQL 权限路由 / 工作台 / Agent 编排

项目定位

项目以“报销系统”为核心入口，向前连接业务申请与审批流程，向后衔接付款、凭证生成、总账查询和财务档案管理，并进一步引入 Agent 工作台，把 Prompt、模型、工具和触发器纳入统一界面，形成面向财务场景的智能协同平台。

6+ 关键业务域联动：报销、流程、付款、凭证、档案、Agent

核心价值

  • 把费用申请、报销审批、凭证生成、归档留痕串成同一条业务链路，减少人工反复搬运数据。
  • 把流程模板、表单模板、凭证映射等规则前置为可维护配置，降低后续扩展成本。
  • 通过首页工作台、最近常用模块、待办统计等设计，让财务人员更快进入高频操作。
  • 引入 Agent 工作台，为“查询、通知、定时触发、工具编排、人机交互”预留统一落地点。
  • 在架构治理上同步推进测试与边界整理，保证功能扩展时系统仍可持续演进。

系统能力拆解

报销与费控工作台

支持新建报销、审批处理、OCR识别、发票验真、报销单查询、待办提醒，并通过首页工作台聚合当前最重要的处理事项。

流程与表单设计

内置流程管理、流程设计器、表单设计器，方便把不同费用场景抽象成可复用、可配置的审批流程。

凭证生成与总账衔接

支持模板级贷方策略、费用类型借方映射、待推送单据池与推送结果查询，把报销结果向总账凭证落地。

电子档案与留痕

围绕单据、财务账簿、附件、票据和相关资料做电子化归档，为后续检索、审计与复盘提供完整链路。

Agent 自动化工作台

把 Prompt、模型、工具绑定、触发器、工作流节点编排纳入统一页面，形成财务场景下的自动化实验台，同时预留了本地化部署接口。

权限、通知与下载中心

通过权限路由、通知角标、下载中心等基础能力增强系统可用性，保证不同角色进入各自的正确入口。
